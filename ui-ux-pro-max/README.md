# Git/CI/CD 可视化演示页面

这是一个现代化的、交互式的Web界面，用于可视化演示Git工作流和CI/CD自动化流程。

## 🎯 功能特点

### Git工作流可视化
- ✅ 实时展示分支结构（main、develop、feature）
- ✅ 提交历史可视化
- ✅ 交互式创建提交和分支
- ✅ Git日志查看

### CI/CD流程可视化
- ✅ 完整的CI/CD管道可视化
- ✅ 实时运行状态展示
- ✅ 多阶段流程演示（Lint、Test、Build、Security）
- ✅ 成功/失败状态反馈

### 统计信息
- ✅ 项目文件统计
- ✅ 测试用例统计
- ✅ 覆盖率显示
- ✅ CI运行次数

## 🚀 快速开始

### 直接打开
直接用浏览器打开 `index.html` 文件即可：

```bash
cd ui-ux-pro-max
open index.html  # macOS
# 或
xdg-open index.html  # Linux
```

### 本地服务器（推荐）
使用Python内置服务器：

```bash
cd ui-ux-pro-max
python3 -m http.server 8000
```

然后访问：http://localhost:8000

## 📖 使用说明

### 1. Git工作流演示

#### 创建新提交
- 点击 **"✨ 创建新提交"** 按钮
- 系统会自动创建一个新的提交
- 新提交会出现在main分支顶部
- 自动触发CI流程

#### 创建分支
- 点击 **"🌿 创建分支"** 按钮
- 系统会创建一个新的feature分支
- 可以点击"合并"按钮将分支合并

#### 查看Git日志
- 点击 **"📜 查看Git日志"** 按钮
- 查看所有提交的详细历史记录

### 2. CI/CD流程演示

#### 运行CI流程
- 点击 **"▶️ 运行CI流程"** 按钮
- 观察各个阶段的执行过程：
  - 🔍 Lint & Format - 代码质量检查
  - 🧪 测试 - 运行单元测试
  - 📦 构建 - 构建验证
  - 🔒 安全扫描 - 安全检查

#### 查看详情
- 点击 **"📊 查看详情"** 按钮
- 查看CI运行的详细信息

#### 重置流程
- 点击 **"🔄 重置"** 按钮
- 重置所有状态，准备下次演示

### 3. 交互式演示建议

#### 演示场景1：完整的开发流程
1. 点击"创建分支" → 创建feature分支
2. 点击"创建新提交" → 在feature分支上提交代码
3. 自动触发CI流程 → 观察CI运行
4. CI通过后，点击"合并" → 合并到main分支

#### 演示场景2：CI流程详解
1. 点击"运行CI流程"
2. 逐步讲解每个阶段的作用
3. 展示成功/失败状态
4. 查看详细的运行日志

## 🔧 自定义配置

### 修改Git数据
在 `script.js` 中修改 `gitData` 对象：

```javascript
const gitData = {
    main: [
        { hash: 'xxx', message: '你的提交信息', time: '时间' }
    ],
    // ...
};
```

### 修改CI阶段
在 `script.js` 中修改 `pipelineState.stages`：

```javascript
pipelineState.stages = ['lint', 'test', 'build', 'security', 'deploy'];
```

### 自定义样式
修改 `styles.css` 中的CSS变量：

```css
:root {
    --primary-color: #2563eb;
    --success-color: #10b981;
    /* ... */
}
```

## 💡 演示技巧

1. **提前准备**：在演示前先运行一次CI流程，确保页面正常工作
2. **逐步讲解**：按照从左到右、从上到下的顺序讲解
3. **互动演示**：鼓励观众提问，实时操作展示
4. **结合代码**：可以同时打开GitHub仓库，展示真实的CI运行
