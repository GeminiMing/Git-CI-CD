# CI/CD演示项目规范

## 项目简介
这是一个用于演示Git和CI/CD基础实践的示例项目。

## Git工作流

### 分支策略
- `main`: 生产环境代码
- `develop`: 开发主分支
- `feature/*`: 新功能分支
- `fix/*`: Bug修复分支

### 提交规范
使用约定式提交（Conventional Commits）：
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `test`: 测试相关
- `refactor`: 重构
- `chore`: 构建/工具

示例：
```
feat(calculator): 添加平方根功能
```

### 提交前检查
```bash
npm run lint
npm test
npm run format:check
```

## 代码质量标准

### 测试要求
- 新功能必须包含测试
- 测试覆盖率不低于80%
- 运行测试：`npm test`

### 代码规范
- ESLint检查必须通过：`npm run lint`
- Prettier格式化：`npm run format`
- 所有公共函数需要JSDoc注释

## 开发流程

### 本地开发
```bash
# 安装依赖
npm ci

# 运行测试
npm test

# 代码检查
npm run lint

# 格式化代码
npm run format

# 运行项目
npm start
```

## CI/CD流程

### 自动化检查
所有提交会自动执行：
1. ESLint代码检查
2. 单元测试
3. 代码格式化检查
4. 构建验证

### 部署
- `develop`分支 → 测试环境
- `main`分支 → 生产环境

## 与Claude协作

### 请求代码生成时
1. 明确功能需求
2. 要求同时生成测试
3. 遵循项目代码规范

### 代码审查清单
- [ ] 通过Lint检查
- [ ] 测试通过
- [ ] 测试覆盖率达标
- [ ] 遵循提交规范
